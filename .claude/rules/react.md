# React 関連のコーディング規約


## メッセージ
### 記載場所
画面に表示する全てのメッセージは、コンポーネント内にハードコーディングせず、`client/message` フォルダ内の YAML ファイルに記述してください。
現状対応している言語は以下の通りです。

- 日本語 — `ja.yml`
- 英語 — `en.yml`
- エスペラント — `eo.yml`

YAML 内の最上位のキー名は、そのコンポーネントが定義されているフォルダ名に揃えてください。

YAML ファイルに定義されたメッセージをコンポーネント内で利用したい場合は、`useTrans` フックを用います。
`useTrans` フックは、`trans`, `transNode` などのフォーマット用関数を返します。
この関数にサブキー名 (さらに階層構造がある場合は `.` で繋げたもの) を渡すと、対応するメッセージを取得することができます。

```typescript
const {trans, transNode} = useTrans("folderName");

trans("foo")      // folderName.foo のメッセージが取得できる
trans("bar.baz")  // folderName.bar.baz のメッセージが取得できる
```

### 数値のレンダリング
画面に数値を表示する際は、必ず `useTrans` フックが返す `transNumber` 関数を介するようにしてください。
これにより、数値が適切にフォーマットされます。

```typescript
const {transNumber} = useTrans("folderName");

transNumber(1000)  // 「1,000」のようにフォーマットされる
```

YAML ファイル内のメッセージ内に数値を含める場合は、以下のように `number` フォーマットを利用してください。

```yaml
foo: "残り {count, number} 問"
```

### テキストの表示に関する注意
本リポジトリでは UI 部品は `zographia` を用います。
長めのテキストや、ユーザー入力など長くなり得るテキストを表示する際は、`zographia` の該当コンポーネント (`SingleLineText`, `MultiLineText`) を利用してください。

## コンポーネント内の関数
コンポーネント内に関数を定義する場合は、原則として全て `useCallback` で囲んでください。

また、コンポーネントの状態に依存しない関数は、できるだけコンポーネントの外側で定義してください。
その場合、コンポーネントの外側に定義する関数は、コンポーネント定義の下に記述してください。
